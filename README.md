# Redrob Ranker — Intelligent Candidate Discovery

**Hackathon:** INDIA.RUNS — Data & AI Challenge  
**Challenge:** Intelligent Candidate Discovery & Ranking  
**Participant:** Anusha Puppala

---

## What this does

Ranks 100,000 candidates against a job description the way a great recruiter
would — not by keyword matching, but by understanding who genuinely fits.

Two-phase hybrid pipeline:
- **Phase 1 (offline):** Pre-computes sentence-transformer embeddings and
  structured features for all 100K candidates. Runs once, saves artifacts.
- **Phase 2 (online):** Loads artifacts and ranks in <5 minutes on CPU,
  with no network access. Combines semantic similarity, rule-based fit scoring,
  and behavioral signal multipliers.

## Setup

```bash
# 1. Clone and enter repo
git clone https://github.com/YOUR_USERNAME/redrob-ranker.git
cd redrob-ranker

# 2. Create virtual environment
python -m venv venv
source venv/bin/activate        # Mac/Linux
venv\Scripts\activate           # Windows

# 3. Install dependencies
pip install -r requirements.txt

# 4. Place the dataset
# Copy candidates.jsonl into data/
# data/sample/sample_candidates.json is already committed

# 5. Sanity check
python scripts/sanity_check.py
```

---

## Reproduce the submission

```bash
# Step 1: Pre-compute embeddings and features (run once, ~20 min)
python scripts/precompute.py

# Step 2: Rank and generate submission (run in <5 min, CPU only, no network)
python scripts/rank.py --candidates data/candidates.jsonl --out outputs/submission.xlsx
```

## Scoring approach

| Component | Weight | What it captures |
|---|---|---|
| Semantic similarity | 50% | Cosine sim of career history text vs JD embedding |
| Title fit | 30% | AI/ML role vs consulting/unrelated, seniority match |
| Experience fit | 10% | Years in JD target band (5-9 yrs) |
| Location fit | 10% | India presence, Pune/Noida preferred, relocation bonus |

*Skills excluded — EDA revealed skills were randomly assigned in the dataset.*

Behavioral multiplier (0.4–1.3×) applied on top: recency, response rate, open_to_work, notice period, GitHub activity.

Honeypot detection via internal consistency checks (expert skills with zero duration, career months exceeding stated YoE).
